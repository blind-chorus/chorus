# How to import this export into the Lovable repo

Target: a local clone of the GitHub repo Lovable created when you connected the project — e.g. `lov-test-v2-bobjobs`. This folder's layout matches that repo's template **1:1**, so the import is a single drag.

---

## The procedure

1. In Lovable, create the project and connect it to GitHub.
2. Clone the generated GitHub repo locally.
3. In Finder, open `lovable-export/` here, **select every entry inside it** (configs, `src/`, `public/`, `docs/`, dotfiles like `.gitignore`/`.prettierrc`), and drag the selection onto the cloned repo's root.
4. When Finder asks, choose **"Replace All"** so renamed/removed chorus files don't leave stale copies behind.
5. In the cloned repo: `git status` to review, `bun install` only if `package.json` changed, then commit and push. Lovable picks up the change on its next sync.

> Drag the **contents** of `lovable-export/`, not the folder itself. Otherwise you'll create a nested `lovable-export/` inside the target repo.

> `.DS_Store`, `IMPORT.md`, and `.lovable/` (chorus-side scratch) are not part of the template — exclude them from the selection, or just delete them in the target repo after the drop.

---

## What gets replaced vs preserved

`lovable-export/` is the chorus-managed surface of the template. Replacing wholesale is safe because:

- **Generated subtrees** (`docs/`, `src/styles.css`, `src/components/chorus/`, `public/*.png`) are wiped and rebuilt on every `npm run build:lovable`, so a wholesale replace mirrors that intent.
- **Hand-maintained baseline** (root configs, `src/routes/`, `src/lib/`, `src/hooks/`, router/server/start files) is kept byte-equivalent to the Lovable template here, so replacing them doesn't lose Lovable-side functionality.

Things the Lovable editor adds outside this tree — additional routes, components under `src/components/` **outside `chorus/`**, lockfile updates — are not in `lovable-export/`, so the drag won't touch them.

Watch-outs:

- **`package.json`** is replaced wholesale. If Lovable added deps, diff first and merge by hand.
- **`src/routeTree.gen.ts`** is auto-generated by TanStack Router. If Lovable added routes, let it regenerate after the import.

---

## Notes

- `npm run build:lovable` is idempotent: re-running it produces a byte-identical tree if no source changed.
- The pre-commit hook + CI stale check ensure this folder is never out of date with `schema/` + `packages/ui/`.
